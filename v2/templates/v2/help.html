Overview

This timecard hour calculator totals time intervals worked on different charge numbers (project codes, WBS codes, or any identifier). It detects breaks between entries and flags invalid or overlapping time ranges.

Totals are rounded to the nearest tenth of an hour. When rounding individual charge numbers would cause them to not add up to the correct rounded total, small adjustments are distributed — preferring the charge number closest to rounding the other way, then the one with the most hours. This ensures the breakdown always adds up exactly and no time is lost or over-reported.

Input Format

Each line starts with an identifier (charge number), followed by a space, then one or more dashed time ranges separated by commas. Time ranges within a line must be in chronological order.

Basic examples

id1 8-8:30, 9-9:30, 11-1, 1:42-3:12, 4:12-6
id2 8:30-9, 9:30-11, 1-1:42, 3:12-4:12

Time notation

  • 8, 8:30 — hours and minutes
  • 8.5 — fractional hours (8 hours 30 minutes)
  • 8a, 8am — explicit AM
  • 3p, 3pm — explicit PM
  • 14, 14:30 — 24-hour format

Using fractional hours:

id1 8-8.5, 9-9.5, 11-1, 1.7-3.2, 4.2-6
id2 8.5-9, 9.5-11, 1-1.7, 3.2-4.2

PM times

Mark PM times explicitly with p / pm or use 24-hour format:

id1 8-12, 4-5
id2 12-4
id3 6p-9       # 'p' suffix marks PM
id1 8-12, 4-5
id2 12-4
id3 18-21      # 24-hour format

Calculation Methods

The calculator runs four methods across two versions (V1 and V2), each using an ordered or unordered interpretation:

  • Ordered — assumes input lines are sorted by first start time. If a line's first time is earlier than the previous line's, PM is automatically inferred.
  • Unordered — assumes PM times are explicitly marked with p, pm, or written in 24-hour format. Only the first time in a line needs to be marked if it is a PM time.

If the methods produce different results, all results are shown and the disagreement is highlighted. Formatting inputs consistently according to one style avoids ambiguity.

Features

Target hour tracking

If you need to hit a certain number of hours for the day, entering a target will calculate what time you need to work until. If you've already met the target, it shows when you hit it.

You can enter a target two ways:

  • Use the Target hours input field below the time entries on the calculator page.
  • Add \= followed by the number on its own line in the time entries text box.
id1 8-5p
\=9.6

Target time: 5:34pm

Inline comments

Add notes to any line — text after #, //, or < is ignored:

id1 8-10   # morning work
id2 10-11  // standup + review
id3 11-6   <remaining tasks>

Leading zeros (24-hour disambiguation)

In ordered mode, a leading zero marks a time as unambiguous 24-hour so PM is not inferred:

id1 11-14, 15:30-17:55
id2 09-11, 14-15:30     # '09' is treated as 9 AM, not inferred as PM

Multi-word identifiers

Charge numbers with spaces are detected automatically — everything before the first time range is used as the identifier:

project alpha 8-10      # ID: "project alpha"
my task: 10-12          # ID: "my task:"

Repeating identifiers

Using the same charge number on multiple lines combines the time ranges:

id1 8-10
id2 10-12
id1 12-1       # combined with the first id1 line; total id1 = 3.0 hrs

Keyboard Shortcuts

Shift + Enter or Ctrl + Enter submits the form without clicking Calculate.

Contact

If you notice a calculation error, please contact the maintainer with the time input that produced the incorrect result.
